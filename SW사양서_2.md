1. 목적
- 관리 대상: requirement, test case, test plan, test execution, defect 추적성/일관성 관리
- 엑셀에 입력된  requirement, test case, test report를 보완해서 프로그램을 통해 RTM에 자동/대량 업로드하고 추적성 링크하기
- requirement, test case, test plan, test execution, defect 에 대해 관리하고 엑셀로 import/export 하여 로컬에서 관리하다가 필요한 이슈는 JIRA RTM에 업로드하기.
- JIRA RTM에 있는 이슈를 조회, 등록, 수정, 삭제하는 기능을 갖기
- JQL을 입력하고 버튼 클릭/엔터키를 통해서 필터링하여 조회하기
- 로컬 db에 JIRA 온라인 데이터를 불러와서 동기화 하거나, 온라인 데이터를 로컬데이터로 덮어쓰는 상향식 동기화, 둘 중하나라도 같고 있는 이슈는 둘 다 갖도록 최대항목으로 동기화하는 방법 중에 골라서 동기화 할 수 있어야 한다.
- 로컬 데이터와 온라인 데이터를 각각 트리에서 선택하면 세부 창에 내용이 각각 조회되어 비교할 수 있도록 한다.
- 로컬에서 직접 데이터를 입력하거나 엑셀에서 입력하고 import 하고 그것을 조회할 수 있어야 한다. 
- JIRA RTM에서 관리되는 데이터(필드)와 동일한 데이터로 엑셀로 관리할 수 있도록 데이터 구조를 설계하고, 그것을 반영하여 export하는 엑셀 파일에 반영해야 한다. export한 파일과 동일한 형태로 작성된 데이터를 import하여 로컬 db에 저장하고, 사용자가 원하는 경우 JIRA RTM에 선택된 데이터를 자동/대량 업로드 할 수 있어야 한다.
- 온라인 데이터의 트리, 세부 내용을 등록/수정/삭제하면 JIRA에 그대로 반영되어야 한다.
- Dashboard에 JQL 등을 통해 원하는 조건대로 필터링을 하여 보여주는 기능을 포함하자.
- Test Case Execution 의 내용으로 구성된 pptx/html/xlsx 양식 보고서로 출력할 수 있는 기능을 구현하자.


2. UX/UI
- 로컬 관리 창, 온라인 데이터 관리 창 별도로 존재
- 서로 비교할 수 있도록 디폴트로는 위/아래로 배치(위:온라인 데이터, 아래: 로컬 데이터)하나 마우스를 통해서 윈도우 위치 배치를 할 수 있도록 구성
- 각 윈도우는 좌/우로 다시 구분하고, 좌측은 트리형태로 구성(폴더)하고 그 하위에는 이슈 배치할 것. 폴더 및 이슈 각각 조회/등록/수정/삭제 기능 구현 할 것. 우측은 선택한 이슈의 세부 정보를 보여주는 세부 내용 창으로 구성한다. 좌와 우의 내용끼리는 마우스를 통해서 폭을 조정할 수 있도록 구성한다.
- 트리는 RTM과 최대한 동일한 컨셉으로 구현하고, 트리구조에 key와 summary까지 표시할 수 있도록 한다.
- 상위 레벨의 각 탭은 6개로 구성된다. Dashboard, Requirements, Test-Cases, Test-Plans, Test-Executions, Defects. 그 중 RTM에서 관리되는 이슈 5개는 RTM의 UX/UI와 최대한 유사하게 구사한다.
- 조회하는 내용의 양에 따라 동적으로 표시 창의 크기를 변경하도록 구현
- JIRA RTM은 Test Key 등의 식별자를 통해서 관리되도록 한다.
